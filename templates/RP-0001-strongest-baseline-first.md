# RP-0001: Strongest Baseline First

## Rule

Before making any comparative claim, identify and evaluate the strongest existing baseline that addresses the same problem.

## Why This Matters

Claims of superiority collapse when compared against weak baselines. Reviewers will immediately identify the strongest missing baseline and reject the contribution.

## Learned From

[Sources will be added as this rule is used in practice.]

## How to Apply

1. Identify the object or property being compared.
2. Search for systems that optimize the same property.
3. Rank baselines by strength on the target property.
4. Compare against the strongest, not the most convenient.
5. If the strongest baseline weakens the claim, narrow or reframe the claim.

## Failure Mode

Do not demand comparison against systems that target a different object or property. The baseline must be a legitimate alternative for the same use case.
