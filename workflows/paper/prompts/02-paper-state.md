# Stage 2: Paper State
# Assess current paper state — claims, argument flow, writing quality.

## Inputs
- `contract.yaml`
- Target paper with unit anchors
- `units/<paper>.units.yaml`
- Project knowledge — claims, terminology

## Task
Produce `paper-state.yaml`:
- Per-section assessment: claims made, argument quality, writing quality
- Claim inventory: all claims with evidence status
- Argument flow: does the paper tell a linear story?
- Writing quality: does it match target venue style (SOSP/OSDI)?
- Positioning: is contribution clear? Related work distinguished?
